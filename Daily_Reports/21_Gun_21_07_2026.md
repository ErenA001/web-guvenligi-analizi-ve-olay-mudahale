# Gun 21 - Severity Genisletme + NVIDIA API Baglantisi

Gun 20'de Flask dashboard temel route ile calisir hale gelmisti. Bugun iki ayri is yapildi: severity seviyesi CRITICAL eklenerek 4'e cikarildi, ayrica ilerleyen gunlerde eklenecek AI ozelligi icin NVIDIA API baglantisi kuruldu.

## Yapilanlar

- config.py'a SEVERITY_HIGH_MAX = 19 sabiti eklendi (mevcut SEVERITY_LOW_MAX ve SEVERITY_MEDIUM_MAX yaninda).
- scoring.py'deki get_severity() fonksiyonuna yeni bir dal eklendi: artik LOW/MEDIUM/HIGH'in ustune CRITICAL de donebiliyor (score 20 ve uzeri).
- Proje icin venv'e python-dotenv ve openai kutuphaneleri kuruldu.
- .env dosyasi olusturuldu (NVIDIA_API_KEY icin), .gitignore'a eklendi. .env.example (bos sablon) de eklendi, bu GitHub'a gidebilir.
- Yeni bir services/ klasoru acildi. Icine services/ai_test.py yazildi: sadece NVIDIA API baglantisini test eden basit bir script, dashboard'a henuz entegre degil.

## Not

- Severity testinde: 192.168.1.11 (score 45) ve 192.168.1.30 (score 28) artik CRITICAL olarak geldi, daha once ikisi de HIGH'di. 10.0.0.9 (score 12) hala HIGH kaldi (yeni ust sinir 19). Diger IP'lerin severity'si degismedi.
- NVIDIA API testinde: services/ai_test.py calistirildi, deepseek-ai/deepseek-v4-pro modeline istek atildi, "Baglanti basarili" cevabi alindi.
- Class kullanilmadi, magic number eklenmedi (yeni sabit config.py'a girdi).

## Sonuc

Severity artik 4 seviyeli, mevcut IP'lerin sonuclari dogru sekilde CRITICAL'a yukseldi. NVIDIA API baglantisi calisir durumda, API key .env'de guvenli sekilde tutuluyor. Sonraki gun: incident type bazli filtreleme.
