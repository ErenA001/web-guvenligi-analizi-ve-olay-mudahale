# Gun 20 - Flask Dashboard'a Gecis

Gun 18-19'daki saglamlastirma asamasi tamamlanmisti, bugun Flask dashboard'a gecis karari verildi (zaman bazli analiz alternatifi ileri bir gune birakildi). Mevcut mimariye (7 modul) dokunmadan uzerine ince bir web katmani eklendi.

## Yapilanlar

- log_analyzer.py'de analyze_logs() fonksiyonu artik hesapladigi IP bazli sonuclari (ip, request_count, score, severity, incident_type, recommendation) bir liste olarak return ediyor. Fonksiyonun geri kalani (print, rapor yazma) degismedi.
- log_analyzer.py'nin en altindaki dogrudan analyze_logs(log_file) cagrisi if __name__ == "__main__": bloguna alindi. Boylece dosya baska bir yerden import edildiginde (app.py gibi) otomatik calisip gercek raporu ezmiyor.
- Proje kokunde yeni app.py dosyasi eklendi: Flask kuruldu, tek route (/) tanimlandi, scripts/log_analyzer.py'den analyze_logs() import edilip cagriliyor, donen veri template'e gonderiliyor.
- templates/index.html eklendi: IP, Request Count, Score, Severity, Incident Type, Recommendation sutunlu duz bir HTML tablo, class/CSS framework yok.
- Proje icin venv (sanal ortam) kuruldu, Flask oraya kuruldu (macOS'ta sistem geneline pip kurulumu engelli oldugu icin gerekti). venv/ klasoru .gitignore'a eklendi.

## Not

- Once eski usul terminal calistirmasi (python3 scripts/log_analyzer.py) tekrar yapildi, davranis birebir ayni kaldi: 43 satir okundu, 2 atlandi, ayni IP'ler ayni severity/incident_type ile geldi.
- Sonra python3 app.py ile Flask calistirildi, http://127.0.0.1:5000 adresinde tablo goruntulendi. Ornekler: 192.168.1.11 -> BRUTE_FORCE, score 45, HIGH. 10.0.0.9 -> FORBIDDEN_ACCESS, score 12, HIGH. 192.168.1.30 -> UNAUTHORIZED_ACCESS, score 28, HIGH. Iki scanner IP (198.51.100.77, 198.51.100.90) ve iki path traversal IP (203.0.113.44, 203.0.113.99) dogru sekilde listelendi.
- Terminal ve web ciktisi arasinda hicbir tutarsizlik yok, ayni veri iki farkli yerde gosteriliyor.
- Class kullanilmadi, magic number eklenmedi, config.py'a dokunulmadi (bu gun icin gerek yoktu).
