# Gün 4 - 25.06.2026

## Konu
IP Bazlı Trafik Analizi

## Günün Amacı
Bugünün amacı, log dosyasındaki IP adreslerini analiz ederek hangi IP adresinden kaç istek geldiğini tespit etmektir.

Bu çalışma, ilerleyen günlerde yapılacak saldırı tespiti, brute force kontrolü ve risk sınıflandırması için temel hazırlık aşamasıdır.

## Yapılan Çalışmalar
- logs/sample_access.log dosyası güncellendi.
- Basit log formatı kullanıldı.
- Python ile log dosyası satır satır okundu.
- Her satırdaki IP adresi ayrıştırıldı.
- IP adreslerine göre request sayısı hesaplandı.
- HTTP status code dağılımı çıkarıldı.
- Sonuçlar terminal ekranında gösterildi.

## Kullanılan Log Formatı
Bu gün için basit bir log formatı kullanıldı:

IP METHOD PATH STATUS

Örnek log satırları:

192.168.1.10 GET /home 200
192.168.1.11 POST /login 401
10.0.0.5 GET /admin 403

## Teknik Açıklama
scripts/log_analyzer.py dosyasında Python kullanılarak log dosyası okundu.

Her log satırı boşluk karakterine göre parçalandı. İlk alan IP adresi, dördüncü alan ise HTTP status code olarak alındı.

IP adresleri defaultdict yapısı ile sayıldı. Böylece her IP adresinden kaç request geldiği hesaplandı.

Ayrıca status code değerleri de sayılarak log dosyasındaki genel trafik durumu gözlemlendi.

## Çıktı
Script çalıştırıldığında şu bilgiler elde edildi:

- Toplam okunan log satırı
- Atlanan satır sayısı
- IP bazlı request sayıları
- HTTP status code dağılımı

Test sonucunda en fazla isteğin 192.168.1.11 IP adresinden geldiği görüldü.

## Öğrenilenler
- Log dosyalarının Python ile nasıl okunacağı öğrenildi.
- IP adreslerinin log satırlarından nasıl ayrıştırılacağı görüldü.
- IP bazlı trafik analizi mantığı anlaşıldı.
- HTTP status code değerlerinin güvenlik analizi için önemli olduğu fark edildi.

## Sonuç
